### 具体词法规则

1. 标识符（ID）
构成规则：必须以字母开头，后续可跟任意字母或数字。
正规式：letter (letter | digit)*
示例：a、b、num123、testVar
说明：本实验未区分关键字与标识符，int、float等均被识别为普通标识符

2. 整型常量（INT）
构成规则：由纯数字序列组成。
正规式：digit+
示例：0、123、999
支持负数形式：-123（由减号后接整型常量构成）

3. 浮点型常量（FLOAT）
    本实验支持三种标准浮点格式：
    1. 小数形式：digit+ . digit*，如123.45、66.
    2. 小数点开头形式：. digit+，如.45、.9
    3. 科学计数法形式：(digit+ | digit+.digit* | .digit+) (e|E)(+|-)? digit+，如1e-3、12.3E+4
正规式：整合上述三种形式的浮点数正规式
示例：123.45、.66、1e-3、9.8E7

4. 界符与运算符
    1. 分号（SCO）：仅识别符号;，作为语句结束标记
    2. 赋值符（ASSIGN）：仅识别符号=，用于变量赋值
    3. 加法运算符（PLUS）：仅识别符号+，不支持++、+=等复合形式
    4. 减法运算符（MINUS）：仅识别符号-，可表示负数或减法运算
    5. 小数点（DOT）：仅识别符号.，用于构造浮点数

#### 空白符处理

源程序中的空格、换行符、制表符仅作为单词分隔符，词法分析器会自动跳过，不生成任何 Token

#### 不支持的词法单元

本实验为基础教学版词法分析器，不支持以下符号与语法：

1. 关键字：if、else、while、int、float等无特殊识别；
2. 乘除运算符：*、/、%；
3. 比较 / 逻辑运算符：>、<、==、&&、||、!；
4. 括号与花括号：(、)、{、}；
5. 复合运算符：++、--、+=、-=等；
6. 字符串 / 字符常量、注释符号。

### 词法分析的前缀

NFA存在歧义，需要维护状态集，速度慢，开销大。词法分析要求线性扫描一次遍历，DFA在 O(n) 时间内完成，无回溯、占用内存小无开销。程序设计语言的词法规则明确，无二义性，因此天然适合用 DFA 识别。

### 主文法的词法类别列表

    ID = L([a-z][a-z0-9]*) \ KEY
    KEY = {int, float, void, if, else, while, return, input, print}
    INT = {int}; FLOAT = {float}; VOID = {void}; IF = {if}; ELSE = {else}
    WHILE = {while}; RETURN = {return}; INPUT = {input}; PRINT = {print}
    NUM = L([+-]?[0-9]+)
    FLO = L([+-]?[0-9]*.[0-9]+ | [+-]?[0-9]+.[0-9]*)
    OP = AOP ∪ ROP ∪ BOP ∪ {=, +=, ++} AOP = {+, -, *, /}
    ADD = {+}; SUB = {-}; MUL = {*}; DIV = {/}
    ROP = {<, <=, ==, >, >=, !=}
    BOP = {∧, ∨, !} 或 BOP = {&&, ||, !}
    ASG = {=}; AAS = {+=}; AAA = {++}
    DLI = {(, ), [, ], \{, \}, \, :, ;}
    LPA = {(}; RPA = {)}; LBK = {[}; RBK = {]}; LBR = {\{}; RBR = {\}}; CMA = {\,};
    SCO = {;}

## 第二部分

程序有两个运行模式：例如：先输入1，再输入符号串个数n，然后输入n个用空格分隔的符号串，程序会给出每个符号串对应的类型，或者先输入2，然后任意输入一行语句，程序会对语句进行词法分析，并给出每个单词对应的类型。
